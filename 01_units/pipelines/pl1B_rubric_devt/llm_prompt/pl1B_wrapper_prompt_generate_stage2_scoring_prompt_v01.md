## pl1B_wrapper_prompt_generate_stage2_scoring_prompt_v02

Wrapper prompt: Generate a tightly bounded **Stage 2 scoring prompt** for **dimension evaluation and boundary rule execution**.

This wrapper prompt generates a prompt. It does not score student work.

---

Before this wrapper prompt can execute, the following input artefacts must be provided verbatim.

All artefacts must use the authoritative grading ontology:

- `participant_id`
- `component_id`
- `dimension_id`
- `indicator_id`
- boundary rules

All artefacts must be supplied in full and delimited using `===`.

Required artefacts:

- `<ASSESSMENT_ID>_AssignmentPayloadSpec_v01`
- `CAL_<ASSESSMENT_ID>_<COMPONENT_ID>_Step02_RubricSpec_v01`

If any artefact is missing or inconsistent, the wrapper prompt must produce no output.

---

## Purpose

Generate one reusable **Stage 2 scoring prompt** that performs:

- dimension satisfaction evaluation
- cross-dimension indicator interpretation
- boundary rule execution
- final performance level assignment

Stage 2 consumes **indicator evidence generated by Stage 1**.

Stage 2 performs **all interpretive scoring logic**.

This wrapper prompt performs:

- prompt synthesis
- rubric constraint propagation
- Stage 2 scoring prompt specification

This wrapper prompt does **not** perform:

- rubric modification
- rule invention
- indicator invention
- coaching or pedagogical advice

## Required Stage 2 Scoring Semantics

The generated scoring prompt must enforce:

- one evaluation per `(participant_id × component_id)`
- dimension satisfaction determined using rubric **indicator–dimension mappings**
- deterministic boundary rule execution
- explicit evaluation of the hardest boundary rule
- explicit reporting of which rule triggered the final score

Dimension satisfaction must be computed from Stage 1 indicator evidence.

Example pattern:

```
Dimension D1 satisfied if indicators I1 AND I2 are present
Dimension D2 satisfied if indicators I3 AND I4 AND I5 are present
Dimension D3 satisfied if indicator I6 is present
```

These mappings must be taken **exactly from the rubric specification**.

---

## Required Stage 2 Evaluation Procedure

The generated scoring prompt must enforce the following evaluation sequence:

1. Retrieve all Stage 1 indicator evidence for the `(participant_id × component_id)` unit.

2. Determine dimension satisfaction using rubric indicator–dimension mappings.

3. Retrieve cross-dimension indicator results.

4. Evaluate rubric boundary rules.

5. Determine which rule condition is satisfied.

6. Assign the corresponding performance level.

Boundary rule evaluation must be **deterministic**.

---

## Ambiguity Handling

If rule application is ambiguous:

- assign the lowest valid performance level
- include flag `needs_review`

---

## Failure Mode Handling

If any required artefact is missing, inconsistent, or contradictory:

- produce no output
- wait silently for corrected inputs
